## 2. 자산 카테고리 정의

| 카테고리 키 | 표시명 | 부호 | 비고 |
|------------|--------|------|------|
| `stock` | 주식/투자 | + | 국내·해외 주식, ETF, 펀드 포함 |
| `real_estate` | 부동산 | + | 호기별 분리 등록 가능 (1호기, 2호기 …) |
| `cash` | 현금 | + | 예금, 적금, 저축 포함 |
| `pension` | 퇴직금 | + | 퇴직연금(DC/DB) |
| `gold` | 금 | + | 실물 금, 금 ETF |
| `crypto` | 암호화폐 | + | 비트코인, 이더리움 등 |
| `loan_mortgage` | 담보대출 | − | 주택담보, 전세대출 |
| `loan_credit` | 신용대출 | − | 신용대출 |
| `loan_minus` | 마이너스 통장 | − | 한도 대비 사용액 |
| `other` | 기타 | + | 분류 외 자산 |

> 대출 3종은 **부채**로 자동 처리 → 총 자산 = 자산 합계 − 부채 합계 (= 순자산)

## 3. 목표 설정

### 3-1. 연간 목표 (올해)
```
목표 이름   : ex) "2026년 3억 달성"
목표 금액   : ₩300,000,000
목표 연도   : 2026  ← 현재연도 고정
```
- 메인 화면 상단에 **올해 목표 전용 배너** 별도 표시
- 연초 자산 대비 현재 자산 증가액 + 남은 목표까지 거리 표시
- 연말까지 남은 개월 수 기준 **월 목표 증가액 자동 계산**
  ```
  월 필요 증가액 = (목표 - 현재 자산) / 남은 개월 수
  ```
- 달성 시 🎉 축하 배너 + 마일스톤 기록

#### 연간 목표 표시 UI
```
┌─────────────────────────────────────────────────────┐
│  🎯 2026년 목표: ₩3억                               │
│  현재 ₩2.76억  ████████████████░░░░░░  92%         │
│  달성까지 ₩2,396만 남음 · 남은 8개월 · 월 ₩299만 필요│
└─────────────────────────────────────────────────────┘
```

### 3-2. 장기 목표 (다년도)
```
목표 이름   : ex) "10억 달성"
목표 금액   : ₩1,000,000,000
목표 연도   : 2030
```
- 여러 목표 등록 가능 (단계별: 5억 → 10억 → 20억)
- 목표별 달성률 게이지 표시
- **현재 성장률 기준 달성 예상 연도 자동 계산**

### 3-3. 목표 달성 예측 로직
```
연평균 성장률(CAGR) = (현재 자산 / 최초 기록 자산)^(1/경과년수) - 1
예상 달성 연도 = 현재연도 + log(목표/현재) / log(1 + CAGR)
```

### 3-4. DB 구조
```sql
-- user_asset_goals 테이블에 goal_type 컬럼 추가
goal_type  text NOT NULL CHECK (goal_type IN ('annual', 'longterm'))
-- annual   : 연간 목표 (target_year = 올해)
-- longterm : 장기 목표 (target_year = 미래)
```

## 4. 월별 스냅샷 구조

현재 `user_assets`는 항목별 row 구조라 월별 집계 이력 관리에 부적합.  
**`user_asset_snapshots`** 테이블을 신규 추가해 월별 전체 자산을 1 row로 저장.

### 스냅샷 입력 방식
- 매월 기준일(사용자 설정, 기본 말일)에 현재 값 입력
- 전월 값 자동 불러와 변경분만 수정하는 UX

### 표시 형태 (스프레드시트 스타일)

| 날짜 | 총 자산 | 주식/투자 | 부동산 | 현금 | 퇴직금 | 금 | 암호화폐 | 대출 | 전월 증가율 | 비고 |
|------|---------|-----------|--------|------|--------|-----|----------|------|------------|------|
| 2024.01 | 267,507,734 | 132,011,409 | - | 99,351,003 | 36,100,000 | - | - | - | +3.69% | |
| 2024.02 | 276,039,815 | 130,961,682 | - | 108,282,782 | 36,750,000 | - | - | - | +3.19% | |

### 집계 통계 (자동 계산)
- **전월 자산 증가율**: `(이번달 - 전월) / 전월 × 100`
- **전년 동월 대비 증가율**: `(이번달 - 작년 같은달) / 작년 같은달 × 100`
- **연 평균 성장률 (CAGR)**: 전체 기간 기준
- **총 누적 증가액**: 최초 기록 대비

## 8. 추가 추천 기능

### A. 대출 관리 고도화
- 대출별 이자율, 만기일, 월 상환액 입력
- 상환 진행률 표시
- **이자 비용 월별 집계** (얼마를 이자로 내고 있는지)
- 대출 상환 시뮬레이션 (추가 상환 시 만기 단축 계산)

### B. 자산 배분 비교
- 권장 포트폴리오 vs 실제 비율 비교 차트
- 연령대별 권장 비율 제안 (예: 40대 = 부동산 40% 이하, 주식 50% 등)

### C. 순저축률 트래커
- 가계부 모듈과 연동
- `(월 총 수입 - 월 총 지출) / 월 총 수입` = 저축률
- 저축률 목표 설정 + 달성 여부

### D. 연간 자산 리뷰
- 매년 1월 전년도 전체 리뷰 리포트 자동 생성
- 연간 자산 증가액, 달성 마일스톤 정리

### E. 환율 연동 (선택적)
- 해외 주식/ETF 보유 시 USD 금액 입력 → KRW 자동 환산

## 9. DB 스키마 추가 ✅ 구현 완료

> **마이그레이션 파일**: `supabase/schema_my_ai_v3.sql` (v4, v5 포함)

### 실제 구현 차이점
- `total_assets`, `total_debt`, `net_worth` → **Generated Column**으로 DB에서 자동 계산 (쿼리 필요 없음)
- `user_asset_goals.description` 컬럼 추가 (v4, 목표 상세 설명)
- `user_asset_milestones.label` 컬럼 추가 (화면 표시용 문자열)
- `user_asset_milestones.goal_id` FK 추가 (사용자 목표 달성 시 연결)
- `user_assets` `asset_type` CHECK 제약 확장 (v5): `pension`, `gold`, `loan_mortgage`, `loan_credit`, `loan_minus`, `other` 추가
- `user_asset_goals.sort_order` 컬럼 추가 (목표 정렬)
- 연간 목표는 **연도당 1개**만 허용 (API 레벨 검증)
- 마일스톤에 `20억` 추가 (`👑 20억 달성!`)

```sql
-- 월별 자산 스냅샷 (핵심 신규 테이블)
CREATE TABLE public.user_asset_snapshots (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid REFERENCES public.users(id) ON DELETE CASCADE NOT NULL,
  snapshot_date   date NOT NULL,               -- 기준일
  stock           numeric DEFAULT 0,           -- 주식/투자
  real_estate     numeric DEFAULT 0,           -- 부동산 합계
  cash            numeric DEFAULT 0,           -- 현금
  pension         numeric DEFAULT 0,           -- 퇴직금
  gold            numeric DEFAULT 0,           -- 금
  crypto          numeric DEFAULT 0,           -- 암호화폐
  loan_mortgage   numeric DEFAULT 0,           -- 담보대출 (부채)
  loan_credit     numeric DEFAULT 0,           -- 신용대출 (부채)
  loan_minus      numeric DEFAULT 0,           -- 마이너스 통장 (부채)
  other           numeric DEFAULT 0,           -- 기타
  note            text,                        -- 비고
  detail          jsonb DEFAULT '{}',          -- 부동산 호기별 상세 등 확장용
  created_at      timestamptz DEFAULT now(),
  UNIQUE(user_id, snapshot_date)
);

-- 자산 목표 (연간 + 장기)
CREATE TABLE public.user_asset_goals (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       uuid REFERENCES public.users(id) ON DELETE CASCADE NOT NULL,
  goal_type     text NOT NULL CHECK (goal_type IN ('annual', 'longterm')),
                -- annual   : 연간 목표 (올해, ex. 2026년 3억)
                -- longterm : 장기 목표 (ex. 2030년 10억)
  name          text NOT NULL,
  target_amount numeric NOT NULL,
  target_year   int NOT NULL,
  is_achieved   boolean DEFAULT false,
  achieved_at   date,
  created_at    timestamptz DEFAULT now()
);

-- 마일스톤 기록
CREATE TABLE public.user_asset_milestones (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       uuid REFERENCES public.users(id) ON DELETE CASCADE NOT NULL,
  milestone_key text NOT NULL,                 -- '1억', '3억', '5억', '10억', 'goal'
  amount        numeric NOT NULL,              -- 달성 시점 자산
  achieved_at   date NOT NULL,
  created_at    timestamptz DEFAULT now()
);
```
